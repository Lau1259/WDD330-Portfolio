The window Object

In short the window object is the global object in a browser. What does this mean? It means that it holds a lot of information and anything created in the global scope belongs to this object.

What information can you get?

  • User agent: This is a reference to the user agent which can tell you which browser you are running on and on which operating system. It is important to note that this is deprecated, but still supported in all major browsers.
  • URL Information: The URL information is stored in the window.location object. This object has properties such as href, protocol, and host. They all provide more information about the URL. An interesting property is the search property that will give you a string with any parameters from the URL.
  • Browser History: You can get the browser history getting the window.history property. Information here includes the amount of pages visited before arriving at the current page. You can also use the go method to go to a specific page in your history.
  • Screen Information: The window.screen object has information about the screen the browser is displayed on. It can be used to find the height and width of the screen in pixels. You can also use the availWidth and availHeight properties to know what space is left on the screen after accounting for system menus.
  • The document object: This is an object that has the properties and methods that deal with the current page. The write method in this object allows a user to write to a page or replace the document if it has already been loaded.

What are workers?

Workers are a way to handle multi-threading in JavaScript. They allow a program to perform complex operations without interrupting the main application.

Useful Modules

  • Nodemon: Nodemon is a good way to run your code. It restarts a server whenever you make changes and can sometimes offer better error messages.
  • Express JS: This is part of both the MERN and MEAN stacks. This module allows you to create a server and listen on specific ports. It can also be used to enable routing on a server.
  • EJS: EJS is a templating engine and serves to mix HTML and JS together. It allows you to pass in data from an express route to the EJS file. That JavaScript data may then be used in the template to create dynamic HTML based on the values passed in by the JavaScript.
  • Node JS Core Modules: These modules include the path, fs, http, and more. These tools each serve a different purpose, but make it easier to work with Node JS and can help with things such as reading and writing files, getting the directory path, and making http request.
  • SASS: This is a CSS preprocessor and I can use it to write SASS or SCSS that will later be convert to CSS. THis allows for nesting and other programing logic like variables and for loops in CSS.
